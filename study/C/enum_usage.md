# 列挙型 (enum)

## enumの基本
`enum`（列挙型）は、プログラム内で使われる意味のある数値（定数）に対して、人間にとってわかりやすい「名前」をつけるための機能です。

### マジックナンバーの排除
プログラム内で突然 `256` や `257` といった数字が現れると、「その数字が何を意味するのか」がコードを読む人（あるいは未来の自分）にはわかりません。これを「マジックナンバー」と呼びます。

`enum` を使うことで、マジックナンバーを意味のある名前に置き換えることができます。

```c
// ❌ 良くない例：マジックナンバー
if (state == 256) {
    // 256が何を意味するのか分からない
}

// ⭕️ 良い例：enumの使用
if (state == Match) {
    // マッチ状態であることが一目でわかる
}
```

## `nfa_cpu.c` での enum の使われ方

今回のNFA（非決定性有限オートマトン）エンジンでの実装では、状態(`State`の`c`)を以下のような `enum` で定義しています。

```c
enum
{
    Match = 256,
    Split = 257,
    Any   = 258,    /* 追加：ワイルドカード */
};
```

### なぜ 256番からスタートするのか？
通常の文字コード（ASCIIなど）の1バイト文字は `0` 〜 `255` の範囲の数値で表されます。  
そのため、NFAの中で「通常の文字（'a' など）」と「特別な制御を行うためのメタ状態」を同じ一つの変数（`int c`）で一緒に扱えるようにするために、意図的に **通常の文字コードには存在しない256番以降** の数値を特別な状態に割り当てています。

* **`Match` (256)**: 状態機械が最後まで到達した「受理（成功）状態」
* **`Split` (257)**: 正規表現の `|`（OR）や `*`（繰り返し）などを実現するための、分岐状態（ε-遷移）
* **`Any` (258)**: 正規表現の `.`（なんでもOK）を実現するための、文字種を見ずに進むワイルドカード状態

このように enum を定義しておくと、以後のプログラム内では `Match` と記述するだけで、コンパイラが自動的に `256` に置き換えて処理を行ってくれます。これにより、「256文字目」「257文字目」という架空の文字を作り出し、アルゴリズムをシンプルに保つことに成功しています。

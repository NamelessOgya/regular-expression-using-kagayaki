# C言語 基本文法リファレンス

C言語のコードを書くうえで頻繁に登場する基本的な文法や、初学者がつまずきやすい記法についてまとめています。

## 1. マクロ定義 (`#define`) の文法

マクロ定義は、コードの中の特定の文字列を、コンパイルする前にパッと別の文字列に一斉置換（コピペ）する仕組みです。プログラムの処理の指示ではないため、**末尾にセミコロン（`;`）がつかない**のが大きな特徴です。

```c
// 基本的な書き方: #define [置き換えたい名前] [置き換え後の内容]
#define MAX_LINE_LENGTH 10000

// このように書いておくと、コンパイラはコード内のすべての MAX_LINE_LENGTH を 
// ひそかに 10000 にコピペ置換してからプログラムの翻訳を始めます。
```

## 2. 関数の定義の文法

関数（一連の処理のまとまり）を実際に作る際の書き方です。インプット（引数）を受け取り、アウトプット（戻り値）を返すのが基本の形となります。

```c
// 戻り値の型  関数名( 引数の型 引数名, ... ) {
//     処理内容;
//     return 戻り値;
// }

// 具体例：2つの整数を受け取って、足し算した結果を整数で返す関数
int add_numbers(int a, int b) {
    int result = a + b;
    return result;
}
```
* **戻り値がいらない場合**: 「空っぽ」を意味する `void` というキーワードを戻り値の型に使います（例：`void show_message() { ... }`）。

## 3. ポインタ (`*`) の文法と意味

C言語で最も特徴的かつ、最もつまずきやすいのが `*`（アスタリスク）を使った**ポインタ**です。
変数や関数名の前についている `*` は、「実体そのもの」ではなく「それがコンピュータのメモリ上のどこにあるのか（**メモリアドレス/住所**）」を扱っていることを示しています。

### ここが厄介！ `*` の2つの顔
コードの中に出てくる `*` は、**書かれる場所によって意味が変わります**。

**① 変数を作るとき（宣言時）の `*`** ＝ 「住所」を入れる箱ですよというマーク
```c
int num = 100;
int *p = &num;  // numの実体の「住所（&）」を、ポインタ箱 p に記憶させる
                // (※ 型の横の * は、「これはポインタ変数を指す」という目印)
```

**② 使うときの `*`** ＝ 住所の「中身（実体）にアクセスする」マーク
```c
int result = *p;  // pに記憶された住所へ行き、その「中身(100)」を取ってくる！
```

### なぜ関数名の前に `*` がつくのか？
`NFA *nfa_compile(const char *regex);` のような記述のことですね。
これは、**関数の戻り値が「実体」ではなく「番地（住所）」である**ことを意味しています。

* **なぜ実体をそのまま返さないの？**: NFA（巨大な正規表現データの塊）などはサイズが非常に大きく、そのままやり取りするとコピー作業が重くなり速度が落ちてしまいます。そのため、メモリ上の広いスペース（ヒープ領域など）にデーン！と実体を作り、関数からは**「〇〇番地に作っておいたから、そこにアクセスしてね！」という住所だけ**を受け渡します。これにより、どんなに巨大なデータでも「住所を渡すだけ」で一瞬でやり取りが完了する仕組みになっています。

## 4. よく使うデータ型まとめ

C言語では関数の戻り値や変数の前に必ず「型（データの種類）」を指定する必要があります。プロジェクトでよく登場する代表的な型は以下の通りです。

### 基本的な型
* **`int` (Integer)**: 整数。`-10` や `0`、`100` などの普通の数字を扱います。
* **`char` (Character)**: 1バイトの文字（半角1文字）。`'A'` や `'x'` などを扱います。
  * `char *` や `char[]` のように配列やポインタとして文字を並べることで、C言語では**「文字列」**として表現します（例: `"hello"`）。
* **`size_t`**: 「サイズ」や「個数」を表すことに特化した特別な数値型です。マイナスになることが絶対にない（符号なし）ため、配列の長さや要素数を数える際によく使われます。

### 特殊な型
* **`void`**: **「何もない（空っぽ）」**を意味する特殊な型です。
  * `void remove_trailing_newline(char *s);` のように関数の最初につくと、「この関数は処理を行うだけで、**何も数値を返しませんよ**」という意味になります。
  * `void *` というポインタとして使われた場合のみ、「なんの型が入っているかは分からない、とりあえずの住所」というさらに特殊な意味になります。
* **自分で作った型 (`struct` など)**:
  * プロジェクト固有で作成したデータ型です。このプロジェクトでは `NFA` や `State` などがこれに該当します。

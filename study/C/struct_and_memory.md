# 構造体とメモリ管理（ポインタ、初期化、確保）

正規表現エンジンの `State` （状態ノード）の実装を通じて学んだ、C言語の構造体とメモリ周りの重要な文法・ベストプラクティスをまとめました。

## 1. 構造体（struct）と型の再定義（typedef）

複数の変数をひとまとめにして「箱」を作る機能が構造体です。

```c
typedef struct State State; // 1. 「struct State」を「State」と省略して呼ぶルールを定義

struct State
{
    int c;           // 条件文字
    State *out;      // 次の状態へのポインタ（矢印1）
    State *out1;     // Splitの場合のもう一方のポインタ（矢印2）
    int lastlist;    // 管理用変数
};
```
* **typedef**: `struct State` と毎回書くのは長いため、`typedef` を使って単に `State` だけで呼び出せるようにエイリアス（別名）を作成しています。
* **箱の役割**: プログラムの実行中に、この箱の中に判定条件や行き先の矢印（メモリアドレス）を格納し、それらを数珠繋ぎにすることでNFAグラフを構築します。

## 2. 構造体の初期化：指示付き初期化子（C99）

構造体のインスタンス化（実体作成）には、モダンで安全な書き方があります。

```c
// ⭕️ 現代の安全な書き方（指示付き初期化子）
State matchstate = {
    .c        = Match,
    .out      = NULL,
    .out1     = NULL,
    .lastlist = 0
};

// ❌ 古い書き方
State matchstate = {Match, NULL, NULL, 0};
```
* `.変数名 = 値` という書き方（指示付き初期化子）を使用することで、「誰に何を入れているか」がコード上で一目瞭然になります。
* 変数の順番が入れ替わっても対応できるため、バグを防ぐ効果があり、現在推奨されているモダンな構文です。

## 3. アロー演算子（`->`）と ドット（`.`）の違い

構造体の中の変数にアクセスするための記号は２つあり、「実体を持っているか」「住所（ポインタ）しか持っていないか」によって使い分けます。

* **ドット演算子（`.`）**
  * 構造体の「実体」が手元にある場合に使います。
  * `matchstate.c`
* **アロー演算子（`->`）**
  * 構造体の「住所（ポインタ）」しか持っていない場合に使います。
  * `State *s` というポインタ変数に対して `s->c` と書くことで、**「sが指し示す先の箱を開けて、その中にある c にアクセスする」** という動きをします。
  * 本来 `(*s).c` と書かなければならない文法のショートカット記号です。

## 4. 動的メモリ確保：malloc と calloc の使い分け

新しく作られた State 箱のためにメモリを確保するシーンです。

```c
State* state(int c, State *out, State *out1)
{
    State *s;              // C89の伝統的スタイル：変数は処理の前に一番上でまとめて宣言する
    
    nstate++;
    s = malloc(sizeof *s); // mallocでメモリを確保
    
    // 確保直後に全変数を自分で初期化（上書き）する
    s->lastlist = 0;
    s->c = c;
    s->out = out;
    s->out1 = out1;
    
    return s;
}
```

* **calloc**: メモリを確保したあと、自動で中身をゼロクリア（`0` で綺麗に）してから渡してくれる関数。
* **malloc**: メモリを確保するだけで、中身は以前のゴミデータが残った状態。非常に高速。

このコードで `malloc` が選ばれているのは、**「確保した直後に、すべての変数（lastlist, c, out, out1）に対して手動で値を上書き（初期化）しているから」**です。どうせ全上書きするなら、わざわざゼロクリアする `calloc` のお掃除処理は無駄になってしまうため、パフォーマンスの観点から最も速い `malloc` が採用されています。

# C言語におけるポインタでの文字列ループ処理

C言語のコード（特にパーサーや低レイヤー処理）を読むと、以下のような少し見慣れない `for` 文が頻繁に登場します。

```c
for (char *current_char = re; *current_char != '\0'; current_char++) {
    // 文字に対する処理
}
```

これは配列の長さ（インデックス）を使わずに、**「文字列を先頭から最後まで1文字ずつスキャンする」** ためのC言語特有のイディオム（定石）です。

## 前提：C言語の文字列と「ヌル文字（\0）」
JavaやPythonなどの文字列とは異なり、C言語の文字列には「自分自身の長さを知っている」という機能がありません。ただの文字の羅列です。
その代わり、文字列の最後には必ず **`\0`（ヌル文字）** という「ここが終端である」ことを示す特殊な番兵がこっそり置かれています。
例: `"abc"` という文字列は、メモリ上では `['a', 'b', 'c', '\0']` となっています。

## ループの3つのステップの解剖

`for ( A ; B ; C )` の各ブロックを紐解くと、ポインタがどう動いているかがよく分かります。

### 【A: 初期化】スタート地点にカーソルを置く
`char *current_char = re;`
* `re`（文字列の先頭アドレス）を、`current_char` というポインタ変数に代入します。
* これで、カーソル（指差し確認の指）が文字列の一番最初の文字にピタッと置かれた状態になります。

### 【B: 条件】番兵にぶつかるまで続ける
`*current_char != '\0';`
* アスタリスク `*` は、「そのポインタが今指している場所の**中身**を見る」という命令です（デリファレンスと呼びます）。
* つまり、「いま指差している場所の文字が `\0`（番兵）じゃない限り、ループを続ける」という意味になります。
* これにより、事前に文字列の長さを測る関数（`strlen`）などを呼ばなくても、安全に最後まで処理し切る事ができます。

### 【C: 更新】カーソルを右に1つ進める
`current_char++`
* ループの処理が1周終わったら、ポインタ自身に `+1` をします。
* こうすることで、ポインタが「隣のメモリ番地」を指すようになり、カーソルが右へ1文字分だけズレます。
* （※C言語ではポインタに+1すると、その型（ここではchar）のサイズ分だけ賢く移動してくれます）

---

## この記法のメリット
普段よく使う `for (int i = 0; i < strlen(re); i++)` というインデックスベースの処理でも同じことはできますが、
ポインタを使ったループ方式は **「いちいち文字の長さを事前に計算しなくてよい」「メモリの場所を直接見て処理するため動作が最速」** という強力なメリットがあるため、低レイヤー実装では圧倒的にこちらが好んで使われます。

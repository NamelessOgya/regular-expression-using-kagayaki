# 正規表現エンジンの基礎用語解説

このドキュメントでは、C言語やその他低レイヤー実装で正規表現エンジンを作る際によく使われる「構文解析（パース）」や「オートマトン（状態遷移）」に関する専門用語をわかりやすく解説します。

## 1. 演算とデータに関する用語

### 演算子 (Operator)
文字データに対して操作やパターン条件を指定するための記号や命令のことです。
* **正規表現における例**:
  * `*` (0回以上の繰り返し)
  * `+` (1回以上の繰り返し)
  * `?` (0回または1回)
  * `|` (OR条件：どちらか一方)
  * `.` または `#` (文字の連結・結合を示す内部的な記号)

### オペランド / 被演算子 (Operand)
演算子によって「操作される対象となる純粋なデータ」のことです。
* **正規表現における例**:
  * `a`, `b`, `c` などの純粋な文字や探したい対象そのもの。
  * 特殊な記号として解釈されず、ただのデータとして扱われる文字です。
  * `a*` の場合、オペランドは `a`、演算子は `*` になります。

### アトム (Atom)
正規表現の構文解析（パース）中によく出てくる用語で、**「ひとつのカタマリとして扱えるこれ以上分割できない最小単位」**のことです。
* 単なる1文字（上記のオペランド）も「アトム」です。
* `(a|b)` のようにカッコで括って計算順序が確定したグループも、全体で1つの「アトム」として扱われます。
* コンパイラの内部では、「アトム（カタマリ）」が2つ並んだ瞬間に間に「連結記号(`#`)」を挟む、といった処理（`num_atom` のカウント）で活躍しています。

## 2. 構文の記法（パース・読み取り方）

人間が書く数式や正規表現は、コンピュータがそのまま解釈するには非常に複雑です（カッコの中身を優先するなどルールがあるため）。そこでエンジン内部では、プログラムが左から右へ1文字ずつ読むだけで一切迷わずに処理を組み立てられる形に変換してから実行します。

### 中置記法 (Infix Notation)
私たち人間が普段書く、**オペランドとオペランドの「間」に演算子が来る**一般的な書き方です。
* 例（数式）: `1 + 2`
* 例（正規表現）: `a | b` または `a(b|c)`

### 後置記法 / 逆ポーランド記法 (Postfix Notation / Reverse Polish Notation)
コンピュータの「スタック（データを上に積んでいくメモリ構造）」と非常に相性が良く、**オペランドを先にまとめて書き、演算子が「最後(後ろ)」に来る書き方**です。
カッコを使わなくても計算の優先順位が完全に確定するという強力なメリットがあります。

* 例（数式）: `1 2 +` 
  * プログラムの動き: `1`を積む → `2`を積む → `+`が来たので、積んである2つを取り出して足す = 3
* 例（正規表現 OR）: `a b |`
* 例（正規表現 連結）: `a b #`
* 例（少し複雑な正規表現）: `a(b|c)*` 
  * [人間の解釈]: `a` の直後に、(`b` または `c`) の繰り返しをくっつける。
  * [コンピュータ向け変換後 (`re2post`関数の出力)]: `a b c | * #`
  * [プログラムの動き]: `a`を積む → `b`を積む → `c`を積む → `|`が来たので `b`と`c`をORでまとめる → `*`が来たので直前のORカタマリを繰り返し状態にする → `#`が来たので最初の`a`と直前の繰り返しカタマリを連結する。

## 3. マッチング・エンジンの仕組み

正規表現を使って文字を検索する際、「もし文字がaだったらこっちの道へ、bだったらこっちの道へ」と状態を移り変えていく仕組みを**オートマトン (自動機械)**と呼びます。

### NFA (Non-deterministic Finite Automaton / 非決定性有限オートマトン)
次に行くべき道（状態遷移）が「複数」同時に存在したり、「ワープ（何も入力文字を消費せずに進む）」できたりする、寄り道や分身が可能な経路図です。
* **特徴**: 分かれ道があった場合、**「プログラム内部で分裂して、全てのルートを並行探索しながら試す」** という力技が可能です。
* **メリット**: 人間が書いた正規表現パターンからNFA経路を作る処理（コンパイル）が非常に早く、使用メモリも少ないです。
* **本リポジトリでの利用**: `nfa_cpu.c` にて使用されています。文字列の中にワイルドカードや「1文字スキップ」などを表現するために利用されています。

### DFA (Deterministic Finite Automaton / 決定性有限オートマトン)
次に行くべき道が、入力文字に対して**絶対に「1つ」に確定している**経路図です。（迷いがない1本道）
* **特徴**: 実行時に「並列で全てのルートを試す」必要がなく、現在の道を進むだけなので検索速度が爆速（処理量が固定）になります。
* **デメリット**: NFAからDFAへ変換（迷いのある分かれ道をすべて塗りつぶしてあらゆる可能性を1本道のグラフに書き出す）する際、パターンによってはグラフの状態数が爆発的に巨大になり（状態爆発）、作成に長い時間がかかったりメモリが枯渇したりする危険性があります。

### Thompson（トンプソン）のアルゴリズム
Ken Thompson氏が1968年に考案した、正規表現をNFAに変換し、それを効率よく安全に検索するアルゴリズムです。本リポジトリのC言語実装に採用されています。
* 分かれ道に出くわすたびに「現在の状態リスト」を分裂させて複数保持し、1文字読むごとに対象となる全ての分裂状態を同時に一歩進める、というアプローチを取ります。
* これにより、正規表現の弱点である「バックトラック（行き止まりになったら前の地点まで戻り直す）」が発生せず、絶対に予測可能な一定時間内で検索が完了する堅牢なエンジンとなります。（※バックトラックを狙ってサーバーをダウンさせる `ReDoS攻撃` などが通用しなくなります）

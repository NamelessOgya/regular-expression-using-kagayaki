# C言語における基本概念まとめ

このドキュメントは、プロジェクトを通じて学んだC言語の基本的な概念や仕組みをまとめた学習ノートです。

## 1. ヘッダーファイル (`.h`) とソースファイル (`.c`) の役割

C言語では、プログラムを部品化して整理するためにファイルの役割を分けます。
* **`.h` ファイル**: どんな関数があるかを宣言する「目次（メニュー表）」
* **`.c` ファイル**: 実際の処理を書いた「実体（レシピ帳）」
他のファイルから機能を利用する際は、設計図である `.h` ファイルを `#include` して使います。

> 📌 **詳細**: 一歩踏み込んだ変数定義のNGルールや、`#pragma once` などのインクルードガードについては、**[ヘッダーファイルの詳細とルール](./header_files.md)** を参照してください。

## 2. コンパイルの実態とリンカの魔法

C言語のコードはそのままでは動かず、コンパイラとリンカと呼ばれるツールを使って「パズルピース（.oファイル）作り」と「パズルの合体」を行う必要があります。

> 📌 **詳細**: オブジェクトファイル(`.o`)の正体や、ファイル階層を無視して関数を呼び出せる「リンク」の魔法については、**[コンパイラとリンカの挙動](./c_compiler_linker.md)** を参照してください。

## 3. メモリの動的確保 (`malloc` と `free`)

C言語はメモリ管理をプログラマ（人間）が直接行う言語であり、それによって高いパフォーマンスを実現しています。

* **`malloc`**: ヒープ領域と呼ばれる場所から、指定したサイズ分のメモリを確保します。NFAのノードなど、実行時に必要な数が決まるデータを保存する際に活躍します。
* **`free`**: `malloc` で確保したメモリは、使い終わったら必ず `free` で解放しなければなりません。これを忘れると**メモリリーク**（メモリの使いっ放し）となり、プログラムが徐々に遅くなったり、クラッシュしたりします。

## 4. ポインタと構造体 (`struct`)

* **ポインタ (`*`)**: 変数の中身そのものではなく、その変数がコンピュータのメモリ上の「どこにあるか（住所）」を示す概念です。配列をやり取りしたり、複雑な NFA のノード同士を矢印（ポインタ）で繋いだりするために欠かせません。
* **構造体 (`struct`)**: 関連する複数の変数をひとまとめにして、新しい「データ型」を作ることができる機能です。オブジェクト指向プログラミングにおけるクラス（プロパティ）の原始的な形とも言えます。

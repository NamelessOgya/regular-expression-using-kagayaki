# C言語における基本概念まとめ

このドキュメントは、プロジェクトを通じて学んだC言語の基本的な概念や仕組みをまとめた学習ノートです。

## 1. ヘッダーファイル (`.h`) とソースファイル (`.c`) の役割

C言語では機能ごとにプログラムを分割して書く際、一般的に **ヘッダーファイル (`.h`)** と **ソースファイル (`.c`)** の2つをペアにして作成します。

### ヘッダーファイル (`.h`)
* **役割**: 「このモジュールにはどんな機能（関数など）があるか」を宣言する**目次（メニュー表）**。
* **特徴**: 実装（中身の処理）は書かず、「関数の名前」「引数の種類」「戻り値の種類」といった**プロトタイプ宣言**だけを記載します。
* **メリット**: 他のファイルがこの機能を呼び出したいときに `#include "..."` することで、事前にその関数の存在を知ることができ、コンパイルエラーを防ぐことができます。

### ソースファイル (`.c`)
* **役割**: 実際にコンピュータが実行する処理の詳細を書いた**実体（レシピ帳）**。
* **特徴**: ヘッダーファイルで宣言した関数の中身のロジックをここに記述します（例: `utils.c` 内の `now_sec` 関数の計算式など）。

## 2. コンパイルの実態とオブジェクトファイル (`.o`)

C言語のコードはそのままではコンピュータが実行できません。コンパイラ（`gcc`など）を使って機械語に翻訳する必要があります。

1. **プリプロセス**: `#include` などを処理し、`.h` の内容を読み込みます。
2. **コンパイル・アセンブル**: `.c` ファイルごとに機械語に翻訳し、**オブジェクトファイル (`.o`)** を作ります。
   * ※ `.o` ファイルは人間には読めない中間ファイルなので、Git管理からは外す (`.gitignore` に追記する) のが一般的です。
3. **リンク**: バラバラに作られた複数の `.o` ファイルをくっつけ合わせ（リンク）、1つの**実行可能ファイル (`.out` や `.exe` など)** を作成します。

## 3. メモリの動的確保 (`malloc` と `free`)

C言語はメモリ管理をプログラマ（人間）が直接行う言語であり、それによって高いパフォーマンスを実現しています。

* **`malloc`**: ヒープ領域と呼ばれる場所から、指定したサイズ分のメモリを確保します。NFAのノードなど、実行時に必要な数が決まるデータを保存する際に活躍します。
* **`free`**: `malloc` で確保したメモリは、使い終わったら必ず `free` で解放しなければなりません。これを忘れると**メモリリーク**（メモリの使いっ放し）となり、プログラムが徐々に遅くなったり、クラッシュしたりします。

## 4. ポインタと構造体 (`struct`)

* **ポインタ (`*`)**: 変数の中身そのものではなく、その変数がコンピュータのメモリ上の「どこにあるか（住所）」を示す概念です。配列をやり取りしたり、複雑な NFA のノード同士を矢印（ポインタ）で繋いだりするために欠かせません。
* **構造体 (`struct`)**: 関連する複数の変数をひとまとめにして、新しい「データ型」を作ることができる機能です。オブジェクト指向プログラミングにおけるクラス（プロパティ）の原始的な形とも言えます。
